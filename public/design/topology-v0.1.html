IAM control-plane strip — the architectural addition. AWS canonical reference diagrams don't show IAM because it's a global service. Cyntro can't afford that omission — IAM gaps ARE the product. The new strip below the DB tier holds the roles actually attached to this VPC's workloads (resolved via HAS_INSTANCE_PROFILE for EC2s and USES_ROLE for Lambdas). Each role gets the same severity language as the data-plane nodes: carmine halo for high-gap, teal border for clean/remediated, slate-dimmed for stale/orphan. The bottom strip answers "who can do what to my workloads, and how over-privileged are they?" without leaving the topology view.
